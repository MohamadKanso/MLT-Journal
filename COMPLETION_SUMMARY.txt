ARCHITECTURE

DATABASE SCHEMA
───────────────
Single 'trades' table with 39 columns:

Core Fields (11):
  id (PK), created_at, trade_name, date, pair, session, position,
  trade_type, setup, risk_pct, rr, result, profit_pct

Structure Fields (7):
  htf_weak, mtf_weak, bos, htf_pricing, mtf_pricing,
  decisional_extreme, poi_reaction

Entry Fields (7):
  entry_confirmation, rejection_candle, order_flow, conditions,
  trade_confluences, confirmation_risk, entry_model_time

Management Fields (7):
  stop_loss, be_criteria, re_entry, target, news, time_of_trade, weekday

Analysis Fields (8):
  dxy, grade, why_grade, top_down, psycho, review_lesson, breakeven, mt5_ticket

Multi-select fields stored as: "Option1, Option2, Option3"

REST API ENDPOINTS
──────────────────
Journal Management:
  GET    /api/trades               - List with filters (result, session, position, grade, weekday, trade_type)
  POST   /api/trades               - Create new trade
  GET    /api/trades/<id>          - Get single trade details
  PUT    /api/trades/<id>          - Update trade
  DELETE /api/trades/<id>          - Delete trade

Analytics:
  GET    /api/stats                - Aggregated journal statistics

MT5 Integration (Proxied):
  GET    /api/live/account         - Account info (balance, equity, margin)
  GET    /api/live/positions       - Open positions
  GET    /api/live/trades          - Closed trades (90 days)
  GET    /api/live/stats           - MT5 performance stats

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

Issue: "MT5 Bridge not running on port 5001"
  Solution: Start MT5 Bridge: python mt5_bridge.py

Issue: "Address already in use" on port 5000
  Solution: Kill existing Flask process or use different port

Issue: "Trades won't save"
  Solution: Check Flask console for errors, verify database permissions

Issue: "Charts not displaying"
  Solution: Wait 30 seconds for auto-refresh, or ensure ≥1 trade logged

Issue: "Multi-select pills not working"
  Solution: Verify JavaScript enabled, check browser console for errors
